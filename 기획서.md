# Synapse
## Memory-Native AI for Thought & Recall
### Final MVP Product Specification (v1.0)

---

## 1. Product Definition

> **"남긴 건 사라지지 않고, 필요할 때 다시 나온다."**

Synapse는 사용자의 대화를 자동으로 기억하고,
과거의 생각을 현재 맥락에 맞게 재등장시켜주는 로컬 기반 AI 시스템이다.

---

## 2. Problem

### 2.1 저장 ≠ 활용
- 사용자는 정보를 저장하지만 다시 활용하지 못함
- 메모는 쌓이지만 "기억"은 형성되지 않음

### 2.2 저장 비용이 높다
- 구조, 태그, 위치 고민 필요
- 저장 자체가 작업이 됨

### 2.3 AI는 기억하지 않는다
- 채팅은 쉽지만 축적되지 않음
- 개인 지식 그래프가 없음

---

## 3. Core Insight

> 사용자는 정보를 저장하지 않는 것이 아니라,
> **저장해도 다시 찾고 활용할 수 없을 것 같아서 저장하지 않는다.**

> 문제는 저장이 아니라 **재등장 경험(Recall)** 이다.

---

## 4. Solution

### 핵심 원칙

> **저장은 자동화하고, 기억의 생성과 재등장을 설계한다.**

---

## 5. Product Identity

> Synapse는 “기억하는 챗봇”이 아니라
> **생각을 연결해주는 엔진이다.**

---

## 6. Core System Architecture

### Dual Engine 구조

#### 1) Conversation Engine
- 현재 대화 유지
- 빠른 응답

#### 2) Memory Engine
- Graph RAG 탐색
- Concept 추출
- Recall 생성

#### 3) Orchestrator
- Recall 개입 여부 판단
- 타이밍 / 강도 / 개수 제어

---

### Context Separation

- Current Context (현재 대화)
- Global Memory (전체 지식 그래프)
- Injected Context (선별된 Recall)

---

## 7. Core Features

### 7.1 Chat-based Input
- 모든 입력은 채팅
- 저장 개념 없음

---

### 7.2 Automatic Memory Formation

- 텍스트 저장
- Concept 추출
- Graph 연결

---

### 7.3 Graph RAG Retrieval

- 의미 기반 검색
- 관계 기반 확장

---

### 7.4 Recall System

#### Trigger Levels

1) 입력 중 (Ghost Hint)
2) 입력 직후 (Suggestion)
3) 질문 시 (Strong Recall)

---

### 7.5 Hyper-Recall Engine

#### (1) Selection Logic

- Bridge Concept
- Temporal Contrast (1주~1개월 이상)
- Domain Crossing

---

#### (2) Trigger Logic

- Reflection Timing (2~3초)
- Question Detection
- Density Threshold (≥3)

---

#### (3) Delivery

- Suggestion 기반 문장
- Insight 생성 중심

---

### Output Rule

> 최대 1~2개의 강한 연결만 제공

## 16. Orchestrator Rule v1.0 (Attention Control System)

Orchestrator는 단순 검색기가 아니라 사용자의 "주의 집중(Attention)"을 관리하는 시스템이다.

---

### 16.1 Trigger Rules (개입 조건)

#### 1) Explicit Intent (의도적 탐색)
- 질문 형태 (?, 어떻게, 뭐였지)
- 과거 지칭 표현 ("지난번", "그때")

---

#### 2) Graph Density (개념 임계점)
- 현재 입력 개념 ≥ 2개
- 특정 과거 Cluster와 강한 연결 발생

---

#### 3) Input Latency (사고 정체)
- 입력 중 5~7초 정지
→ 도움 필요 신호로 판단

---

#### 4) Cross-Domain Bridge (도메인 교차)
- 서로 다른 영역 간 추상적 유사성 발견
→ Hyper-Recall 트리거

---

### 16.2 Silence Rules (개입 금지 조건)

#### 1) Low Value Input
- 인사, 감탄사
- 단순 명령 (예: 알람 설정)

---

#### 2) Flow State 보호
- 빠른 연속 입력 시
→ 모든 개입 중단

---

#### 3) Recency Filter
- 최근 10분 내 재등장된 내용
→ 재노출 금지

---

#### 4) Ambiguity Filter
- 낮은 Confidence Score
- 파편적 매칭
→ Recall 생성 금지

---

### 16.3 Core Principle

> 필요한 순간에만 개입하고, 나머지 시간은 침묵한다.
